          Pipeline B (cross-asset extension): marginals come from Pipeline A; dependence is added.
====================================================================================================

Pipeline      : B. Cross-asset extension on top of Pipeline A marginals.
Tickers       : SPY, NVDA, JNJ, JPM, AAPL, QQQ
Marginals     : Per-asset CHMM-N (Gaussian emissions, K = 6), trained independently per ticker
                (same fits as the CHMM-N rows of Table T2).
Dependence    : Four competing constructions
                  SIM              : Single Index Model regressing each non-market ticker on SPY.
                  Gaussian copula  : rank-reordering with Kendall-tau-based correlation matrix.
                  Student-t copula : rank-reordering with profile-MLE nu* = 6.0.
                  Truncated C-vine : root-centered pair-copula vine, edge family chosen by AIC.
Market factor : SPY (drives the SIM factor path).
Paths / alpha : 200 simulated paths per dependence model; KS thresholded at alpha = 0.05.

What this table tests:
  Given per-ticker CHMM marginals that already pass univariate fidelity (Table T2), which
  dependence mechanism best reproduces the empirical cross-asset correlation matrix?
  Two metrics are reported: per-asset KS pass rates (sanity check on marginals after
  dependence is injected) and correlation-matrix distance (the core dependence metric).

How this differs from Table T2:
  Table T2 (results/SPY/Table-T2-Per-Ticker-Emission-Families.txt) evaluates the per-ticker
  marginal only, across THREE EMISSION FAMILIES, with NO dependence. Table T3 holds the
  emission family fixed (Gaussian) and varies the DEPENDENCE MECHANISM.
